### /plan

**Purpose:** To create a detailed, step-by-step project plan from a high-level request.

**Usage:** `/plan "[High-level request or problem statement]"`

**Description:**
This is the core command for the Planner Agent. It takes a project goal, a feature request, or a refactoring task and breaks it down into a structured, actionable plan.

The output will include:

- **Objective:** The overall goal of the project.
- **Analysis:** A summary of the problem and potential solutions.
- **Phased Plan:** A breakdown of tasks into small, manageable phases.
- **Success Criteria:** Measurable metrics to determine project completion.
- **Potential Risks & Mitigation Plan:** Identification of potential issues and a strategy to address them.
- **Documentation Plan:** A plan for updating all relevant project documentation.
