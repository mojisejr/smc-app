🔧 1. Communication Overview
Hardware
Interface: RS485 (primary), TCP/IP (via BU module)

Baud rate (default): 19200

Data Bits: 8

Stop Bit: 1

Parity: None

Software Packet Format (Big-endian)

[STX][ADDR][LOCKNUM][CMD][ASK][DATALEN][ETX][SUM][DATA...]
Field Description
STX Start byte: 0x02
ADDR CU12 address 0x00 ~ 0x0F (or 0x10 = broadcast)
LOCKNUM Lock index 0x00 ~ 0x0B (or 0x0C = all)
CMD Command (see command table below)
ASK Usually 0x00 when sending
DATALEN Length of DATA block
ETX End byte: 0x03
SUM Checksum (sum of all bytes including DATA)
DATA Optional payload

🧾 2. Command Reference
Name CMD DATA Format Description
Get Lock Status 0x80 None Query current lock state
Unlock Lock 0x81 None Unlock specified or all locks
Query/Set Unlocking Time 0x82 u16 (in 10ms unit) Default: 550ms (0x0037)
Query/Set Baud Rate 0x83 u8 0:9600, 1:19200, 2:57600, 3:115200
Query/Set Delayed Unlock Time 0x84 u8 seconds Delay before unlocking starts (default: 0)
Query/Set Push Door Wait Time 0x85 u8 seconds Wait for door push detection (default: 0)
Initialize CU12 0x8E None Reset all settings to factory default
Query Software/Hardware Version 0x8F None Returns [softVersion, hardVersion] (e.g., 0x10, 0x12)

🖧 3. RS485 Setup (Serial)
Connection

RJ45 port (or 5-pin XH2.54)

Pin Mapping (CU12 → RS485 Adapter):

A → T/R+

B → T/R-

Y → RXD+

Z → RXD-

GND → GND

Serial Settings

Baud Rate: 19200

Data Bits: 8

Stop Bit: 1

Parity: None

🌐 4. TCP/IP Setup (Optional via BU Module)
Hardware Connection

PC <-> Router <-> BU <-> CU12 via RS485

BU Configuration

Set IP (e.g. 192.168.0.158)

Port: e.g. 5000

Baud rate: must match CU12 (default: 19200)

Communication

Protocol same as RS485 (CU12 protocol)

Send/receive in HEX over TCP socket

✅ 5. Response ASK Value Table
ASK Meaning
0x10 Success
0x11 Failure
0x12 Timeout
0x13 Unknown command
0x14 Checksum error
0x00 Default when sending

⚠️ 6. Notes for Implementation
Always wait >500ms after Initialize, Set Unlocking Time, Set Delay, or Set Push Door Time.

For multiple CU12s on RS485, ensure unique ADDR per device (0x00 ~ 0x0F)

0x10 is broadcast ADDR — use with care.
