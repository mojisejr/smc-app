Step 1.3: Automation Scripts (Copy & Paste)สร้างไฟล์ Script ทั้ง 3 ไฟล์นี้และนำโค้ดด้านล่างไปใส่ นี่คือเครื่องจักรที่จะทำงานแทนคุณ1. scripts/workflow/generate_prompt.sh(สร้าง "ร่าง Prompt" สำหรับทุกรอบ)#!/bin/bash

# Description: Auto-generates draft prompt files for all rounds defined in CLAUDE.md

# Usage: npm run wf:prompt -- all

set -e # Exit immediately if a command exits with a non-zero status.

PROMPTS_DIR="prompts/rounds"
CONTEXT_FILE="context/CLAUDE.md"

echo "🤖 V4 Workflow: Starting Prompt Scaffolding..."

if [ ! -f "$CONTEXT_FILE" ]; then
echo "❌ Error: Master context file not found at '$CONTEXT_FILE'"
exit 1
fi

# Simple parser to find round definitions in CLAUDE.md

# Looks for lines like "### Round X: Round Name"

ROUND_LINES=$(grep -E "^### Round [0-9]+:" "$CONTEXT_FILE")

if [ -z "$ROUND_LINES" ]; then
echo "⚠️ Warning: No round definitions found in '$CONTEXT_FILE'. Cannot generate prompts."
exit 0
fi

echo "Found round definitions. Cleaning old drafts..."
rm -f ${PROMPTS*DIR}/round*\*.md
mkdir -p $PROMPTS_DIR

echo "$ROUND_LINES" | while IFS= read -r line; do
    # Extract Round Number and Name
    ROUND_NUM=$(echo "$line" | sed -n 's/### Round \([0-9]*\):.*/\1/p')
    ROUND_NAME=$(echo "$line" | sed -n 's/### Round [0-9]*: \(.*\)/\1/p')
    CONTEXT_LINE=$(grep -A 1 "### Round ${ROUND_NUM}:" "$CONTEXT_FILE" | tail -n 1)
SUPPLEMENT_FILES=$(echo "$CONTEXT_LINE" | sed -n 's/._Context: `CLAUDE.md` + `\(._\)`/\1/p')

    PROMPT_FILE="${PROMPTS_DIR}/round_${ROUND_NUM}.md"

    echo "📝 Creating draft for Round ${ROUND_NUM}: ${ROUND_NAME}"

    # Create the draft prompt file
    cat > "$PROMPT_FILE" << EOL

# Round ${ROUND_NUM}: ${ROUND_NAME} [AUTO-GENERATED DRAFT]

**CONTEXT**: โปรดอ่านไฟล์ Context ต่อไปนี้อย่างละเอียด:

1. \`context/CLAUDE.md\` (Master Context)
2. \`context/supplements/${SUPPLEMENT_FILES}\` (Round-Specific Context)

**ROUND OBJECTIVE**:
[โปรดตรวจสอบและขยายความ Objective ของรอบนี้ให้ชัดเจน]

**PAIRED TASKS**:

- **Task A**: [อธิบาย Task A ที่ต้องทำ]
- **Task B**: [อธิบาย Task B ที่ต้องทำ]

**SUCCESS CRITERIA**:

- [ ] [เกณฑ์ความสำเร็จข้อที่ 1 ที่สามารถทดสอบได้]
- [ ] [เกณฑ์ความสำเร็จข้อที่ 2 ที่สามารถทดสอบได้]
- [ ] โค้ดทั้งหมดต้องเป็นไปตามมาตรฐานใน \`TECH-SPECS.md\`

**ACTION**:
โปรดดำเนินการตาม Paired Tasks ที่กำหนด และแจ้งความคืบหน้าเมื่อแต่ละ Task เสร็จสิ้น
EOL
done

echo "✅ V4 Workflow: Prompt Scaffolding complete. Drafts are in '${PROMPTS_DIR}'." 2. scripts/workflow/run_test_feedback.sh(รัน Test และสร้าง Debug Prompt หากล้มเหลว)#!/bin/bash

# Description: Runs tests and generates a debug prompt if any test fails.

# Usage: npm run wf:test

set -e

DEBUG_FILE="prompts/DEBUG_PROMPT.md"
echo "🧪 V4 Workflow: Running automated tests..."

# Run the test command, allowing it to fail without exiting the script

TEST_OUTPUT=$(npm test 2>&1)
EXIT_CODE=$?

if [ $EXIT_CODE -ne 0 ]; then
echo "❌ Tests Failed! Generating self-correction debug prompt..."

# Create the debug prompt file

cat > "$DEBUG_FILE" << EOL

# 🚨 DEBUG REQUIRED: Automated Test Failed

**Analysis Request:**
การทดสอบอัตโนมัติล้มเหลว โค้ดที่สร้างขึ้นล่าสุดมีข้อบกพร่อง โปรดวิเคราะห์ Error Log ด้านล่างอย่างละเอียด เปรียบเทียบโค้ดปัจจุบันกับ Requirement ใน \`context/CLAUDE.md\` และไฟล์ Supplement ที่เกี่ยวข้อง จากนั้นดำเนินการแก้ไขโค้ดให้ถูกต้อง

**Error Log:**
\`\`\`
$TEST_OUTPUT
\`\`\`

**Action:**

1.  **Analyze**: ระบุสาเหตุของปัญหาจาก Error Log
2.  **Identify**: ชี้เป้าไฟล์และส่วนของโค้ดที่ต้องแก้ไข
3.  **Implement**: แก้ไขโค้ดเพื่อให้การทดสอบทั้งหมดผ่าน
    EOL

echo "✅ V4 Workflow: Debug prompt created at '${DEBUG_FILE}'."

else
echo "✅✅✅ V4 Workflow: All tests passed successfully!"

fi 3. scripts/workflow/create_commit.sh(สร้าง Commit Message ตาม Template และจัดการ Git)#!/bin/bash

# Description: Creates a standardized git commit for a completed round.

# Usage: npm run wf:commit -- "feat(round-X): your commit message"

set -e

COMMIT_MSG=$1

echo "❌ Error: Commit message is required."

Part 2: The Project Lifecycle (From 0 to 100)เราจะสาธิต Workflow V4 ด้วยการสร้างโปรเจกต์ "Link Shortener" ซึ่งเป็นแอปพลิเคชันระดับ Production ที่สมจริงStep 2.1: The Initial Briefing (Human Task)คุณเริ่มต้นโปรเจกต์โดยการส่ง Briefing นี้ให้ผม (AI)[ตัวอย่าง Initial Briefing สำหรับ Link Shortener]1. PROJECT OVERVIEW: สร้างบริการย่อลิงก์ (Link Shortener) ที่มีความน่าเชื่อถือและรวดเร็ว ผู้ใช้สามารถวาง URL ยาวๆ แล้วระบบจะสร้าง URL สั้นๆ ที่ไม่ซ้ำกันให้ เมื่อมีคนเข้า URL สั้น ระบบจะ Redirect ไปยัง URL เดิม2. USER FLOWS:Anonymous User: เข้าหน้าแรก -> วาง URL ยาว -> กด "Shorten" -> ได้รับ URL สั้น -> คัดลอกไปใช้งานRedirect Flow: ผู้ใช้คลิก URL สั้น -> ระบบค้นหา URL ยาวจากฐานข้อมูล -> Redirect ผู้ใช้ไปยัง URL ยาว3. UI/UX GUIDELINES:ดีไซน์แบบ Minimalist, Single Page.มี Input field ขนาดใหญ่ตรงกลางหน้าสำหรับวาง URL.มีปุ่ม "Shorten" ที่ชัดเจนเมื่อได้ผลลัพธ์ จะแสดงการ์ดที่มี URL สั้น พร้อมปุ่ม "Copy"4. TECHNICAL REQUIREMENTS:Short URL Generation: ต้องสร้าง "slug" (ส่วนท้ายของ URL) ที่สั้น (6-8 ตัวอักษร) และไม่ซ้ำกันDatabase: ใช้ฐานข้อมูลเพื่อเก็บคู่ของ slug และ originalUrlAPI: สร้าง API endpoint สำหรับ:POST /api/links: รับ originalUrl และคืนค่า shortUrlGET /{slug}: (จัดการโดย Next.js Middleware) เพื่อทำ Redirect5.
